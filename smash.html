SMASH - SMALL SHELL


One of the operation system course (234123) projects was to write a simple linux shell.
Our shell has many attrbutes:

  • "chprompt new_prompt": allow the user to change the prompt displayed.
    If no parameters were sent to then the prompt shall be reset to smash, otherwise change the prompt display to the string which we received.
  • "showpid": prints the smash pid.
  • "pwd": prints the full path of the current working directory.
  • "cd new_path": change directory
    - "cd -": change the directory to the last working directory.
    - "cd path": change the directory to the directory which path parameter is its path.
  • "jobs": jobs command prints the jobs list which contains:
    - background jobs.
    - stopped jobs.
  • "kill -signum job ID" :sends a signal which its number specified by "signum" to the job which its sequence ID in jobs list is "job ID".
  • "fg": brings a stopped process or a process that runs in the background to the foreground.
    - "fg job_ID": resume the job with job_ID.
    - "fg": resume the max job in job list.
  • "bg": resumes one of the stopped processes in the background.
    - "bg job_ID": resume the job with job_ID.
    - "bg": resume the last stopped job in job list.
  • "quit": exits the smash.
    - "quit kill": print the jobs which killed before existing.
  • "external": executing external commands that are not part of the built-in commands.
  • "pipe":
    - "command1 | command2": redirects command1 stdout to its write channel and command2 stdin to its read channel.
    - "command1 |& command2": redirects command1 stderr to the pipe’s write channel and command2 stdin to the pipe’s read channel.
  • I/O:
    - "command > output_file.txt": causes the command stdout file-descriptor to be redirected to output-file and writes any output produced by “command” to the given “output-file”.
    - "command >> output_file.txt": will append command output to the given output-file if exists.
    If the output-file does not exist then it creates it and if it exists then it overrides its content.
  • "cp oldPath newPath": copies a file from oldPath which could be a relative or full path to an existing file to a new file specified by the newPath which could be a relative or full path.


We implement our shell using c++ programming language, we create an object for the command which saves the command line, PID, and its state.
Besides, we choose to implement the job list as a vector each item is an object JobEntry which contain a pointer for the command struct, job ID, and the time that the job enters the job list.
